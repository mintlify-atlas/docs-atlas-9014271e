---
title: Update professor
description: Update an existing professor belonging to the authenticated user.
---

`PUT /api/professors/:id`

<Note>
  This endpoint requires authentication. Include a valid Bearer token in the `Authorization` header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the professor to update.
</ParamField>

### Body

<ParamField body="fullName" type="string" required>
  New full name for the professor. Maximum 100 characters.
</ParamField>

## Response

**200 OK** — Returns the updated professor object.

<ResponseField name="id" type="string" required>
  Unique identifier of the professor (UUID).
</ResponseField>

<ResponseField name="fullName" type="string" required>
  Updated full name of the professor.
</ResponseField>

<ResponseField name="userId" type="string" required>
  ID of the authenticated user who owns this professor.
</ResponseField>

## Examples

<RequestExample>

```bash cURL
curl --request PUT \
  --url http://localhost:3000/api/professors/c3d4e5f6-a7b8-9012-cdef-123456789012 \
  --header 'Authorization: Bearer $ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "fullName": "Juan Carlos Pérez"
  }'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "id": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "fullName": "Juan Carlos Pérez",
  "userId": "u1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

</ResponseExample>

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | The request body is missing required fields or contains invalid values. |
| `401 Unauthorized` | No token provided or the token is invalid. |
| `404 Not Found` | No professor with the given ID exists for the authenticated user. |
