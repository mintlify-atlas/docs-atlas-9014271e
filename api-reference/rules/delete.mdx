---
title: Delete rule
description: Permanently delete a scheduling rule by its ID.
---

`DELETE /api/rules/:id`

<Note>
  All endpoints require an `Authorization: Bearer <access_token>` header. Data is scoped to the authenticated user.
</Note>

<Warning>
  This action is irreversible. The rule will be permanently removed from the user's scheduling configuration.
</Warning>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the rule to delete.
</ParamField>

## Response

`204 No Content` — The rule was successfully deleted. No response body is returned.

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | Rule with the given `id` does not exist or does not belong to the authenticated user. |

<RequestExample>

```bash cURL
curl --request DELETE \
  --url http://localhost:3000/api/rules/e5f6a7b8-c9d0-1234-efab-345678901234 \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

</RequestExample>

<ResponseExample>

```json 404
{
  "statusCode": 404,
  "message": "Rule not found",
  "error": "Not Found"
}
```

</ResponseExample>
