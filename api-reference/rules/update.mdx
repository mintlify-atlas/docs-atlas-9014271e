---
title: Update rule
description: Replace all fields of an existing scheduling rule. All body fields are required.
---

`PUT /api/rules/:id`

<Note>
  All endpoints require an `Authorization: Bearer <access_token>` header. Data is scoped to the authenticated user.
</Note>

<Warning>
  This is a full replacement — all body fields must be provided. The `userId` is always taken from the authenticated token and cannot be changed.
</Warning>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the rule to update.
</ParamField>

### Body

<ParamField body="type" type="string" required placeholder="NO_GAPS">
  The rule type. Must be one of the supported rule types. Maximum 50 characters.

  | Type | Parameters |
  | ---- | ---------- |
  | `NO_GAPS` | None |
  | `NO_EARLY_MORNINGS` | None |
  | `NO_LATE_EVENINGS` | None |
  | `PREFER_MORNING` | None |
  | `PREFER_AFTERNOON` | None |
  | `COMPACT_DAYS` | None |
  | `BALANCED_LOAD` | None |
  | `MIN_FREE_DAY` | `daySelect` (optional) |
  | `MAX_CONSECUTIVE_HOURS` | `number` (required, 1–8) |
  | `LUNCH_BREAK_PROTECTED` | `timeRange` (required) |
  | `AVOID_PROFESSOR` | `professorSelect` (required) |
</ParamField>

<ParamField body="priorityOrder" type="number" required placeholder="1">
  Updated priority of this rule. Lower numbers are evaluated first. Minimum value: `1`.
</ParamField>

<ParamField body="parameters" type="object">
  Updated rule-specific parameters. The expected shape depends on the `type`. Set to `null` or omit for rules that do not require parameters.
</ParamField>

## Parameter formats by rule type

<AccordionGroup>
  <Accordion title="MIN_FREE_DAY — daySelect">
    ```json
    {
      "type": "daySelect",
      "value": 5
    }
    ```
    `value` is a day number 1–7 (1 = Monday, 7 = Sunday). Parameter is optional for this rule type.
  </Accordion>

  <Accordion title="MAX_CONSECUTIVE_HOURS — number">
    ```json
    {
      "type": "number",
      "value": 3
    }
    ```
    `value` is the maximum number of consecutive class hours. Accepted range: 1–8.
  </Accordion>

  <Accordion title="LUNCH_BREAK_PROTECTED — timeRange">
    ```json
    {
      "type": "timeRange",
      "value": {
        "start": "12:00",
        "end": "14:00"
      }
    }
    ```
    `value.start` and `value.end` are times in `HH:MM` format.
  </Accordion>

  <Accordion title="AVOID_PROFESSOR — professorSelect">
    ```json
    {
      "type": "professorSelect",
      "value": "b2c3d4e5-f6a7-8901-bcde-f12345678901"
    }
    ```
    `value` is the UUID of the professor to exclude from scheduling.
  </Accordion>
</AccordionGroup>

## Response

`200 OK`

<ResponseField name="id" type="string" required>
  UUID of the rule.
</ResponseField>

<ResponseField name="type" type="string" required>
  Updated rule type.
</ResponseField>

<ResponseField name="priorityOrder" type="number" required>
  Updated priority order.
</ResponseField>

<ResponseField name="parameters" type="object | null">
  Updated rule-specific parameters, or `null` if none.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this rule.
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | Invalid request body (missing `type` or `priorityOrder`, invalid values). |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | Rule with the given `id` does not exist or does not belong to the authenticated user. |
