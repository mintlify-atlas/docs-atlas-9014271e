---
title: Get a saved schedule
description: Retrieve a single saved schedule by its ID.
api: GET /api/saved-schedules/{id}
---

Returns the full details of a saved schedule, including all associated sections. Only schedules belonging to the authenticated user can be retrieved.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

## Path parameters

<ParamField path="id" type="string" required>
  UUID of the saved schedule to retrieve.
</ParamField>

## Response

Returns a single `SavedSchedule` object.

<ResponseField name="id" type="string" required>
  UUID of the saved schedule.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the saved schedule.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the schedule was saved.
</ResponseField>

<ResponseField name="sections" type="object[]" required>
  The sections associated with this saved schedule.

  <Expandable title="section properties">
    <ResponseField name="sectionId" type="string" required>
      UUID of the section.
    </ResponseField>

    <ResponseField name="subject" type="string" required>
      Name of the subject.
    </ResponseField>

    <ResponseField name="professor" type="string | null" required>
      Full name of the professor, or `null` if unassigned.
    </ResponseField>

    <ResponseField name="startTime" type="string" required>
      Start time in `HH:MM` format (24-hour, UTC).
    </ResponseField>

    <ResponseField name="endTime" type="string" required>
      End time in `HH:MM` format (24-hour, UTC).
    </ResponseField>

    <ResponseField name="days" type="string[]" required>
      Days of the week this section meets.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Description |
|--------|-------------|
| `401 Unauthorized` | Missing or invalid Bearer token. |
| `404 Not Found` | No saved schedule with the given ID exists for the authenticated user. |

<RequestExample>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/saved-schedules/a9b8c7d6-e5f4-3210-9876-fedcba543210 \
  --header 'Authorization: Bearer '$ACCESS_TOKEN
```
