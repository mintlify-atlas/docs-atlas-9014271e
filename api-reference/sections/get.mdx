---
title: Get section
description: Retrieve a single section by its ID.
---

`GET /api/sections/:id`

<Note>
  All endpoints require an `Authorization: Bearer <access_token>` header. Data is scoped to the authenticated user.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the section to retrieve.
</ParamField>

## Response

`200 OK`

<ResponseField name="id" type="string" required>
  UUID of the section.
</ResponseField>

<ResponseField name="startTime" type="string" required>
  Start time in `HH:MM` format.
</ResponseField>

<ResponseField name="endTime" type="string" required>
  End time in `HH:MM` format.
</ResponseField>

<ResponseField name="subject" type="object" required>
  The subject this section belongs to.
  <Expandable title="properties">
    <ResponseField name="id" type="string" required>
      UUID of the subject.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Name of the subject.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="professor" type="object | null" required>
  The assigned professor, or `null` if none.
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      UUID of the professor.
    </ResponseField>
    <ResponseField name="fullName" type="string">
      Full name of the professor.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="days" type="string[]" required>
  Names of the days on which this section meets.
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | Section with the given `id` does not exist or does not belong to the authenticated user. |

<RequestExample>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/sections/c3d4e5f6-a7b8-9012-cdef-123456789012 \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

</RequestExample>
