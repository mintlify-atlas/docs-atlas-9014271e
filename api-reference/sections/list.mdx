---
title: List sections
description: Retrieve all sections belonging to the authenticated user.
---

`GET /api/sections`

<Note>
  All endpoints require an `Authorization: Bearer <access_token>` header. Data is scoped to the authenticated user.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

## Response

`200 OK` — Returns an array of section objects. Returns an empty array if none exist.

<ResponseField name="id" type="string" required>
  UUID of the section.
</ResponseField>

<ResponseField name="startTime" type="string" required>
  Start time in `HH:MM` format.
</ResponseField>

<ResponseField name="endTime" type="string" required>
  End time in `HH:MM` format.
</ResponseField>

<ResponseField name="subject" type="object" required>
  The subject this section belongs to.
  <Expandable title="properties">
    <ResponseField name="id" type="string" required>
      UUID of the subject.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Name of the subject.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="professor" type="object | null" required>
  The assigned professor, or `null` if none.
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      UUID of the professor.
    </ResponseField>
    <ResponseField name="fullName" type="string">
      Full name of the professor.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="days" type="string[]" required>
  Names of the days on which this section meets.
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | Missing or invalid bearer token. |

<RequestExample>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/sections \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

</RequestExample>
