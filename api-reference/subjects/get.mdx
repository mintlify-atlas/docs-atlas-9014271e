---
title: Get subject
description: Retrieve a single subject by ID for the authenticated user.
---

`GET /api/subjects/:id`

<Note>
  This endpoint requires authentication. Include a valid Bearer token in the `Authorization` header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token. Format: `Bearer <access_token>`
</ParamField>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the subject to retrieve.
</ParamField>

## Response

**200 OK**

<ResponseField name="id" type="string" required>
  Unique identifier of the subject (UUID).
</ResponseField>

<ResponseField name="name" type="string" required>
  Name of the subject.
</ResponseField>

<ResponseField name="userId" type="string" required>
  ID of the authenticated user who owns this subject.
</ResponseField>

## Examples

<RequestExample>

```bash cURL
curl --request GET \
  --url http://localhost:3000/api/subjects/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  --header 'Authorization: Bearer $ACCESS_TOKEN'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Matemáticas",
  "userId": "u1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

</ResponseExample>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | No token provided or the token is invalid. |
| `404 Not Found` | No subject with the given ID exists for the authenticated user. |
