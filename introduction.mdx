# Scheduly API

Scheduly API is a NestJS REST API that generates optimized academic schedules for university students. It uses a two-phase backtracking algorithm with a rule-based scoring system to find the best schedule combinations based on your preferences.

## What is Scheduly API?

University students often face the challenge of building a semester schedule that fits multiple constraints: avoiding early morning classes, minimizing gaps between lectures, keeping certain days free, or avoiding specific professors. Scheduly API solves this by:

1. Accepting a list of subjects (courses) you need to enroll in
2. Applying your configured preference rules (e.g., no early mornings, compact days)
3. Running a backtracking search over all valid section combinations
4. Returning the **top 3 optimized schedules** ranked by score

## How it works

<Steps>
  <Step title="Register and authenticate">
    Create an account and obtain a JWT access token. All resource endpoints require a valid Bearer token.
  </Step>
  <Step title="Create your data">
    Add the subjects you want to take, the professors who teach them, and the sections (time slots) available.
  </Step>
  <Step title="Configure optimization rules">
    Define rules ranked by priority — for example, avoid early mornings at priority 1, prefer morning classes at priority 2, and no gaps at priority 3.
  </Step>
  <Step title="Generate schedules">
    Call `POST /api/scheduling/generate` with the subject IDs you want. The engine returns the top 3 schedules scored by your rules.
  </Step>
  <Step title="Save your preferred schedule">
    Persist the schedule you like best with `POST /api/saved-schedules` so you can retrieve it later.
  </Step>
</Steps>
